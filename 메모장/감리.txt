해결해야될거
1. 비용문제 (낮을수록 좋음)
2. 데이터셋을 못구하는 특정구역은? -> (선택) HITL 에서 사용자가 지도에 그려서 추가
3.cleaning_ops가 다른 데이터셋에 적용됬을때도 정제된 데이터가 원하는대로 나왔는가 - [STEP2 정제데이터 VS 휴먼 정제 데이터] 로 판단 후 cleaning_ops, audit_ops_catalog.py 수정 (아직 확인 필요)

해결
1. web_search 부정확 <- 여차하면 hitl 페이지에서 사용자가 검색 후 넣게 -> 해결(데이터셋에서 뽑은 region, facility 와 법률 조회 api로 보강)
2. 다른 데이터셋에도 감리Ai 작동하는가 -> 작동확인
3. web_search 검색값 cache에 바로 저장할지 -> web_search만 사용시 비정확한 값이 많았지만 법률 api로 프롬프트 보강해주면서 정확도 상승, 확실치 않은 경우엔 hitl로 확인후 cache에 저장
4. 조례 없을시 낮은 web_search 결과 값 -> 조례 데이터 없으면 바로 hitl 단으로 넘김 ( 비용, 시간 대비 결과 안나옴)
